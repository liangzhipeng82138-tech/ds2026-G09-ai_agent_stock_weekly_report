

## 三、项目目录结构（统一规范）

每个小组的作业压缩包须按以下结构组织：

```
ProjectName/
├── readme.md              ← 项目简介、分工说明、GitHub 链接
├── data_raw/              ← 原始数据（直接获取，不做修改）
├── data_clean/            ← 清洗后的数据
├── output/                ← 图表、报告、结果文件
├── 01_get_data.ipynb      ← 数据获取
├── 02_data_clean.ipynb    ← 数据清洗与整理
├── 03_analysis.ipynb      ← 探索性/验证性分析
└── 04_visualization.ipynb ← 数据可视化（可与 03 合并）
```

> 具体 `.ipynb` 文件名可根据项目内容灵活调整，但须保持编号顺序。  
> E 类题目（Quarto book）目录结构另见各题说明。

## 四、Notebook 写作规范

每个 `.ipynb` 文件的**头部 Markdown Cell** 须包含：

```markdown
# 项目名称 — 模块说明

| 项目   | 内容 |
|--------|------|
| 课程   | 数据分析与经济决策（ds2026） |
| 题目   | T##：XXX |
| 小组   | 第 X 组 |
| 成员   | 姓名（学号）、姓名（学号）…… |
| GitHub | https://github.com/xxx/xxx |
| Pages  | https://xxx.github.io/xxx |
| 日期   | YYYY-MM-DD |
```

**每个分析任务的结构**：

```
[Markdown] 任务说明：本步骤的目标和方法简介
[Code]     实现代码
[Markdown] 结果解读：图表/数据的含义、发现、局限性
```

## 五、GitHub 要求

1. 每组建立一个独立的 GitHub 仓库（建议命名：`ds2026-G##-topicname`），其中 `G##` 是组号，`topicname` 是选题关键词（如 `stock-analysis`）
2. 开启 **GitHub Pages**，将项目报告（或 Quarto 渲染结果）发布为网页
3. 在项目 `readme.md` 头部填写仓库地址和 GitHub Pages 网址
4. 建议使用分支（branch）管理各成员的工作，通过 Pull Request 合并

## 七、评分标准（满分 100 分）

| 维度 | 分值 | 说明 |
|------|------|------|
| 数据获取 | 20 | 数据来源合理、获取代码可复现、有必要的注释 |
| 数据清洗 | 15 | 缺失值处理、格式规范、字段说明清晰 |
| 分析深度 | 25 | 有实质性发现，不止于描述，有初步解释 |
| 可视化质量 | 20 | 图表美观、标题/轴标签/图例完整、传达信息清晰 |
| 报告规范 | 10 | Notebook 结构规范、说明文字完整、GitHub 发布正常 |
| 分工说明 | 10 | readme 中有明确分工，组员贡献可识别 |

## 九、使用 AI 工具的说明

本课程**鼓励**使用 AI 工具（Claude、ChatGPT、Cursor、GitHub Copilot 等），但须遵守以下规则：

1. 在 notebook 头部或 readme 中**注明使用了哪些 AI 工具**
2. 代码必须能在本地/云端**实际运行**，AI 生成的代码须经过测试
3. 分析结论和解读文字**不能全部由 AI 生成**，须体现小组自己的理解
4. 鼓励在 notebook 中记录「向 AI 提问的过程」和「遇到的坑」
