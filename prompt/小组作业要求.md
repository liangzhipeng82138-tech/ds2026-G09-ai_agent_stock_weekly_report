## 一、作业目标

通过完整的小组项目，训练以下核心技能：

- **数据获取**：使用 `akshare`、`baostock`、`yfinance`、`OpenBB`、`FRED API`、`World Bank API`、`GitHub API`、`arxiv` 等 Python 包获取在线数据；或通过简单爬虫、政府网站下载数据
- **数据整理**：清洗、合并、格式转换、构建 SQLite 关联数据库
- **探索性分析**：描述统计、分布分析、相关分析、时序特征
- **数据可视化**：折线图、热力图、箱线图、地图等
- **技术写作与协作**：Quarto 报告/在线书、GitHub 项目管理、AI 辅助工作流

## 三、项目目录结构（统一规范）

每个小组的作业压缩包须按以下结构组织：

```
ProjectName/
├── readme.md              ← 项目简介、分工说明、GitHub 链接
├── data_raw/              ← 原始数据（直接获取，不做修改）
├── data_clean/            ← 清洗后的数据
├── output/                ← 图表、报告、结果文件
├── 01_get_data.ipynb      ← 数据获取
├── 02_data_clean.ipynb    ← 数据清洗与整理
├── 03_analysis.ipynb      ← 探索性/验证性分析
└── 04_visualization.ipynb ← 数据可视化（可与 03 合并）
```

> 具体 `.ipynb` 文件名可根据项目内容灵活调整，但须保持编号顺序。  
> E 类题目（Quarto book）目录结构另见各题说明。

## 九、使用 AI 工具的说明

本课程**鼓励**使用 AI 工具（Claude、ChatGPT、Cursor、GitHub Copilot 等），但须遵守以下规则：

1. 在 notebook 头部或 readme 中**注明使用了哪些 AI 工具**
2. 代码必须能在本地/云端**实际运行**，AI 生成的代码须经过测试
3. 分析结论和解读文字**不能全部由 AI 生成**，须体现小组自己的理解
4. 鼓励在 notebook 中记录「向 AI 提问的过程」和「遇到的坑」

### 高效使用 AI 的提示词模板

**数据获取类：**
```
我在用 Python 的 {包名} 获取 {数据描述}，
请给我一段完整的示例代码，包括安装、导入、调用和结果预览。
```

**报错调试类：**
```
我在运行以下代码时报错：
[贴代码]
错误信息是：[贴报错]
我的 Python 版本是 X.X，{包名} 版本是 X.X，请帮我找出原因并修复。
```

**可视化美化类：**
```
我有一个 DataFrame，列名是 [XXX]，请用 matplotlib/plotly 画一张 {图表类型}，
要求：中文标题、轴标签、图例，配色使用 {风格}，输出为 PNG（300dpi）。
```
